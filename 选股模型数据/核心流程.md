# 选股模型核心流程

## 一、数据获取

### 1.1 数据源
- **AkShare**: `stock_yjbb_em()` (业绩报表) + `stock_zcfz_em()` (资产负债表)
- **股价**: `stock_zh_a_hist()` 缓存 + 增量拉取
- **覆盖**: 全A股 6大类(周期资源/制造/科技/消费/金融地产/公用事业) ~5000只

### 1.2 拉取范围
- 季度: 2019Q1 ~ 最新季
- 每季度拉取后合并 compute_indicators 生成特征

---

## 二、数据清洗

### 2.1 基础过滤
- `buy_price <= 0` → mark as unusable
- `interest_coverage` 全空 → 填 4.0 (全局中位)
- 季度编码修正: month 03/06/09/12 → quarter 1/2/3/4

### 2.2 极端值缩尾 (1%/99% winsorize)
对所有特征列按年缩尾:

| 特征 | 说明 |
|---|---|
| `roe` | ROE |
| `gross_margin` | 毛利率 |
| `net_margin` | 净利率 |
| `profit_yoy` | 利润同比 |
| `asset_turnover` | 资产周转率 |
| `inventory_turnover` | 存货周转率 |
| `liability_to_asset` | 负债率 |
| `current_ratio` | 流动比率 |
| `cfo_to_revenue` | 经营现金流/营收 |
| `cfo_to_profit` | 经营现金流/利润 |
| `pe` | 市盈率 |
| `pb` | 市净率 |
| `ps` | 市销率 |
| `buy_price` | 买入价格 |

`forward_return` 在同年内 1%/99% 缩尾。

### 2.3 缺失值填补
所有特征列 `fillna(median)`。

---

## 三、特征工程

### 3.1 14维输入特征
```
roe, gross_margin, net_margin, profit_yoy, asset_turnover,
inventory_turnover, liability_to_asset, current_ratio,
cfo_to_revenue, cfo_to_profit, pe, pb, ps, buy_price
```

注: `interest_coverage` 已废弃 (消融实验证明全行业下无增益)

### 3.2 辅助特征
- `gm_pct` = 毛利率在 行业-年 内的百分位排名 (用于掩码)
- 6行业 one-hot: `ind_公用事业, ind_制造, ind_周期资源, ind_消费, ind_科技, ind_金融地产`

### 3.3 标签 (L)
```
forward_return < -0.1  → 0 (暴跌)
forward_return < 0     → 1 (跌)
forward_return < 0.1   → 2 (小涨)
forward_return < 0.3   → 3 (中涨)
forward_return >= 0.3  → 4 (暴涨)
```

### 3.4 标准化
训练前 `StandardScaler` 标准化，scaler 随权重保存。

---

## 四、掩码筛选 (S1∩S3∩S4)

仅在**预测/部署阶段**使用，不参与训练。训练集使用全部可用数据。

| 掩码 | 条件 | 说明 |
|---|---|---|
| S1 | `cfo_to_revenue >= 0` | 正向经营现金流 |
| S1 | `current_ratio >= 0.5` | 流动比率不低于0.5 |
| S3 | `gm_pct >= 0.1` | 毛利率不低于行业10%分位 |
| S4 | `liability_to_asset <= 0.8` | 负债率不超80% |

---

## 五、模型

### 5.1 Random Forest
```python
RandomForestClassifier(
    n_estimators=100,
    max_depth=20,
    min_samples_leaf=50,
    class_weight={0:1, 1:1, 2:1, 3:1, 4:0.5},
    n_jobs=-1,
    random_state=42
)
```
- 参数: ~200k
- 优势: 三数据集泛化最稳(全行业+40.8%, 8行业+43.5%, 4行业+36.2%)

### 5.2 REG MLP (256x128x64)
```python
Sequential(
    Linear(20, 256), ReLU, Dropout(0.5),
    Linear(256, 128), ReLU, Dropout(0.5),
    Linear(128, 64), ReLU, Dropout(0.5),
    Linear(64, 5)
)
```
- 优化器: AdamW(lr=0.001, weight_decay=0.02)
- 损失: CrossEntropyLoss(weight=[1,1,1,1,0.5])
- 训练: 5000 epoch, GPU
- 参数: ~47k
- 优势: 全行业年+46.9% (单模型最强)

## 六、训练流程

### 6.1 验证模式 (5折CV)
每年作为测试集，其余4年作为训练集。训练5个权重，逐年均值评估模型性能。

### 6.2 部署模式 (单权重)
全部历史季度(2019Q1 ~ 上一季)作为训练集，训1个权重用于预测最新季。

---

## 七、选股策略

```
优先: REG ∩ RF 共识买入
回退: REG 单独买入 (RF罢工时兜底)
筛选: 所有买入均通过 S1∩S3∩S4 掩码
```

### 7.1 策略表现 (5折CV)

| 策略 | 全行业 | 8行业 | 4行业 |
|---|---|---|---|
| REG∩RF | 223只+57.0% | 26只+70.2% | 24只+35.1% |
| REG单独 | 593只+46.9% | 86只+40.6% | 46只+22.2% |
| RF单独 | 425只+40.8% | 79只+43.5% | 87只+36.2% |

## 九、死路清单

| 实验 | 结果 | 原因 |
|---|---|---|
| w=0.2 暴涨惩罚 | 买0-4只 | 过于保守, 模型拒绝预测暴涨 |
| CART+REG (去Hard+高惩罚) | 4414买+7.7% | 双重正则化饿死信号 |
| LR 线性回归 | 6买+24% | 全行业上完全保守 |
| 128x64 w=0.5 drop=0.2 | 1060买+37.8% | 弱于REG高惩罚版 |
| 季度预测 | 季均-5.1% | 噪声淹没问题 |
| 三模型共识(含LR) | 1买 | LR不可用 |
| interest_coverage 保留 | -2.5pp(4行业) | 全行业下无增益 |
